# steinmetz

This document is currently in flux.  The interface could change at
any time.  (I don't intend this to be a permanent state of affairs.

## Options

Command-line options are described by the Option record type.
The high-level syntactic interface to creating Options will be
described first, followed by the high-level procedural interface,
and then finally the lower-level core procedures exported by
‘(steinmetz options)’.


### Syntactic interface

(options <clause> ...) → (list Option)

The 'options' form is a convenient interface to the options library.
Each clause describes a single option and is of one of the following
forms:

    (flag   <name-or-names> [<help-text>])
    (option <name-or-names> [<arg-name> [<help-text> [<conv>]]])

The (flag …) form describes a boolean flag which takes no arguments;
(option …) describes an option taking an argument.
<name-or-names> is either an identifier or a list of identifiers
and gives the acceptable forms of an option. <arg-name> and <conv> have
the same meaning as they do in make-option. <help-text> is a string
describing the option.

The value of an 'options' form is a list of option structures
representing the CLI options described by its clauses.

Example:

    (define my-opts
      (options
        (option (f file)    FILE "input file")
        (option (k chunk)   NUM  "chunk to operate on" string->number)
        (flag   (v verbose)      "verbose output")))


### Procedural interface

#### High-level

These procedures are exported by ‘(steinmetz parse)’.  They
automatically generate parsers and fill in details which users of the
‘(steinmetz options)’ interface must provide explicitly.


(make-cli-option names [arg-name [conv-proc [properties]]]) → option

Argument types:

(: arg-name (or symbol #f))
(: conv-proc procedure)
(: properties (alist symbol *))

Returns an Option object describing a command-line option.

If *arg-name* is provided, the command-line option will take a (single)
required argument.  If it is false, the option will not take an
argument (and its parser will raise an exception if an argument is
provided).  TODO: Optional arguments.

If *conv-proc* is provided, it must be a procedure which accepts a
single string argument and returns a single value.  *conv-proc* is
applied to the value of the option’s command-line argument.
*conv-proc* defaults to ‘values’.

If *properties* is provided, it must be a symbol-keyed alist.  Any
number of arbitrary properties may be added.  (See also “Common
properties” below.)  *properties* defaults to an empty list.

The parser generated by ‘make-cli-option’ will associate a name of the
option with the value of ‘(*conv-proc* *arg*)’, where *arg* is the
option’s command-line argument, if the option is encountered at all.

(make-cli-flag names [properties]) → option

A restricted version of ‘make-cli-option’ for creating “flags”, options
which take no arguments and are merely present or absent.  *names* and
*properties* have the same types and meanings they do for
‘make-cli-option’.

The parser generated by ‘make-cli-flag’ will associate a name of the
flag with ‘#t’, if the flag is encountered at all.


##### Common properties

Most options include a 'help' property associated with a string
describing the option.


#### Core procedures

(option? obj) → boolean

Returns ‘#t’ if *obj* is an Option and ‘#f’ otherwise.


(option-names option) → (list symbol)

Returns a list of acceptable names for *option*.


(option-argument-name option)

The name (a symbol) of the option's argument, used in documentation
and diagnostic messages.  '#f' if the option takes no argument.


An Option also contains a key-value mapping for arbitrary properties.
These can be manipulated with the following procedures.

(option-property-ref option key) → *

Returns the value associated with *key* (a symbol) in *option*'s
property mapping.  If no association exists, returns #f.  (FIXME:
This should probably raise an exception on failure, to avoid the
ambiguity associated with returning #f.)

(option-set-property option key obj) → option

Returns a copy of *option* with a new property association of
*key* to *obj*.  Any existing association for *key* is replaced.




## Parsing

(parser-exception message irritant ...)

Raises a continuable exception indicating that something went wrong
during command-line parsing.  *message* must be a string.  The
*irritant*s may be anything.  The raised object is a compound condition
satisfying ‘parser-condition?’ and including (at least) ‘&message’ and
‘&irritants’ components.


(parser-condition? obj) → boolean

True of *obj* is a condition raised by ‘parser-exception’.


(process-cli options [cli-list]) -> [alist, list]

A easy-to-use high-level parsing procedure.  *options* is a list of
Option objects.  Parses *cli-list* (a list of strings, or the value of
‘(cdr (command-line))’ by default) and returns two values: an alist
associating the name of each option encountered with its argument, and
a list of tokens unassociated with any of the options (the “operands”).

If an unknown option is encountered while parsing, an object satisfying
‘parser-condition?’ is raised continuably.

Example:

    (let ((opts (options
                  (option (f) FILE "input file")
                  (flag (v) "verbose output"))))
      (process-cli opts '("-f" "myfile" "-v" "frobnitz")))

    ⇒ ((f . "myfile") (v . #t))
      ("frobnitz")
